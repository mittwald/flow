# Grundlagen
Die Notification ist eine Component zur Darstellung von System- oder Statusmeldungen, die überwiegend im Hintergrund ablaufende Prozesse betreffen. Notifications können sowohl durch Aktionen des Users ausgelöst werden, wie etwa das Starten einer Installation, als auch durch systemseitige Ereignisse, beispielsweise eine Warnung über geringen Speicherplatz.  Notifications sollten in einer zentralen Ansicht oder einem Benachrichtigungscenter gesammelt werden, um auch zu einem späteren Zeitpunkt einsehbar zu sein. 

## Best Practices
Achte bei der Verwendung einer Notification darauf, dass…
* nur relevante und wichtige Informationen kommuniziert werden, um Informationsüberlastung zu vermeiden.
* die Meldung kurz, klar und verständlich formuliert ist (Faustregel nicht mehr als 12 Wörter). 
* Notifications gesammelt und zu einem späteren Zeitpunkt abrufbar sind.

## Verwendung
Verwende eine Notification, um…
* den User über den Fortschritt oder Abschluss von Hintergrundprozessen zu informieren.
* systemseitige Hinweise, Warnungen oder Fehler bereitzustellen.
