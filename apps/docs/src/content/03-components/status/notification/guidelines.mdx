# Grundlagen

Die Notification ist eine Component zur Darstellung von System- oder
Statusmeldungen, die überwiegend im Hintergrund ablaufende Prozesse betreffen.
Notifications können durch Aktionen des Users ausgelöst werden, zum Beispiel
durch das Starten einer Installation. Sie können aber auch durch systemseitige
Ereignisse entstehen, wie etwa eine Warnung bei geringem Speicherplatz.
Notifications sollten in einer zentralen Ansicht oder einem
Benachrichtigungscenter gesammelt werden, um auch zu einem späteren Zeitpunkt
einsehbar zu sein.

## Best practices

Achte bei der Verwendung einer Notification darauf, dass ...

- nur relevante und wichtige Informationen kommuniziert werden, um
  Informationsüberlastung zu vermeiden.
- die Meldung kurz, klar und verständlich formuliert ist (Faustregel: nicht mehr
  als 12 Wörter).
- Notifications gesammelt und zu einem späteren Zeitpunkt abrufbar sind.

## Verwendung

Verwende eine Notification, um ...

- den User über den Fortschritt oder Abschluss von Hintergrundprozessen zu
  informieren.
- systemseitige Hinweise, Warnungen oder Fehler bereitzustellen.
