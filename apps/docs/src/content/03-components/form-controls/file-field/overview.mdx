# Playground

Verwende `<FileField />`, um einen FileField darzustellen. Innerhalb des FileFields kommen die Components [Label](/03-components/content/label) und [Button](/03-components/actions/button) zum Einsatz.

---

# Disabled

Ist das FileField **disabled**, ist keine Interaktion möglich. Sobald die Ursache behoben ist, kann sie wieder wie gewohnt verwendet werden.

<LiveCodeEditor example="disabled" editorCollapsed />

---

# Invalid

Bei ungültiger Eingabe wird das FileField invalidiert. Über `<FieldError />` kann eine entsprechende [Fehlermeldung](/02-foundations/03-content-guidelines/02-fehlermeldungen) ausgegeben werden.

<LiveCodeEditor example="invalid" editorCollapsed />

---

# Mit FieldDescription

Um wichtige Hinweise zum FileField bereitzustellen, kann unterhalb eine `<FieldDescription />` eingebaut werden. In vielen Fällen empfiehlt es sich, die erlaubten Dateitypen dort anzugeben.
