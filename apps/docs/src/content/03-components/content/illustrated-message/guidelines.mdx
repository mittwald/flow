# Grundlagen

Die IllustratedMessage dient als Platzhalter, wenn der eigentliche Inhalt nicht angezeigt werden kann. Sie bietet eine Kombination aus [Icon](https://mittwald.github.io/flow/03-components/content/icon), einer [Heading](https://mittwald.github.io/flow/03-components/content/heading) und einer unterstützenden Beschreibung. Optional kann sie durch interaktive Elemente wie [Buttons](https://mittwald.github.io/flow/03-components/actions/button) oder [Links](https://mittwald.github.io/flow/03-components/navigation/link) ergänzt werden. Ziel ist es, leere Seiten visuell aufzulockern und dem User kontextbezogene Hinweise zu geben, wie er fortfahren kann.

# Best Practices

Achte bei der Verwendung einer IllustratedMessage darauf, dass…
* das [Icon](https://mittwald.github.io/flow/03-components/content/icon) zum Kontext der Seite passt und nicht ablenkt.
* der Text klar beschreibt, warum der Zustand leer ist und was als Nächstes zu tun ist.
* Handlungsauforderungen mit interaktive Elemente wie [Buttons](https://mittwald.github.io/flow/03-components/actions/button) oder [Links](https://mittwald.github.io/flow/03-components/navigation/link) unterstützt werden.

# Verwendung

Verwende eine IllustratedMessage, um…
* dem User visuelles und inhaltliches Feedback zu geben, wenn keine Daten vorhanden sind.
* Vorschläge oder Handlungsoptionen aufzuzeigen, die dem User helfen, Inhalte zu erstellen oder zu entdecken.
* gesperrte Bereiche freundlich und verständlich zu gestalten, um zum Beispiel Wartezeiten zu überbrücken.
