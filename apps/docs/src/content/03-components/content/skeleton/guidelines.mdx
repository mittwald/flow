# Grundlagen

## Best practices

Achte bei der Verwendung von Skeletons darauf, dass...

- sie den Inhalt der Benutzeroberfläche so gut widerspiegeln, dass der User erahnen kann, was sich dahinter verbirgt.
- der Detailgrad und die Anzahl der Skeletons nicht zu unruhig oder überladen wirken.
- auf der Benutzeroberfläche keine verwirrende Mischung aus Skeletons und Components im Disabled State entsteht.

## Verwendung

Verwende Skeletons, um dem User bereits während des Ladezustands eine visuelle Orientierung zum Seitenaufbau zu geben. Skeletons eignen sich speziell für das initiale Laden der Seite. Für kurzfristige Ladezustände als Feedback, wie etwa der Pending State beim [Buttons](/03-components/actions/button), sollte hingegen ein [LoadingSpinner](/03-components/status/loading-spinner) verwendet werden.
