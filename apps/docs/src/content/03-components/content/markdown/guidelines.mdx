# Grundlagen
Markdown ist besonders nützlich in Situationen, in denen Inhalte von Usern oder externen Quellen 
geliefert werden. Die Ausgabe wird als HTML gerendert, jedoch ohne explizite Kontrolle über 
semantische Elemente wie `<p>`, `<h1>`, etc. Für mehr Kontrolle über das Layout sollte 
[Text](/03-components/content/text) verwendet werden.


## Best practices

Achte bei der Verwendung von Markdown darauf, dass sie hauptsächlich für dynamische oder nutzergenerierte Inhalte (z. B. im [Chat](/03-components/chat/chat)) genutzt wird.


## Verwendung

Verwende Markdown, um Inhalte, die von Nutzern im Markdown-Format verfasst wurden (z. B. in Kommentarfunktionen oder Chats), korrekt darzustellen.
