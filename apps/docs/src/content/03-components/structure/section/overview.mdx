## Sub-Headings

Sections beginnen in der Regel mit einer
H2-[Heading](/03-components/content/heading/overview). Wenn eine feinere
Unterteilung nötig ist, können Headings auf niedrigerem Level direkt in der
Section platziert werden. Zur optischen Verdeutlichung erhalten Sub-Headings in
der Größe H3 automatisch einen größeren Abstand zum darüberliegenden Content.

# SectionAction

Die Section verfügt in der rechten oberen Ecke über einen Bereich für Actions –
die sogenannte SectionAction. Dort können ein
[Link](/03-components/navigation/link/overview), ein
[Switch](/03-components/form-controls/switch/overview), ein oder zwei
[Buttons](/03-components/actions/button/overview) oder eine Kombination dieser
Components verwendet werden.

<LiveCodeEditor example="link" editorCollapsed />

<LiveCodeEditor example="switch" editorCollapsed />

Es können maximal zwei [Buttons](/03-components/actions/button/overview)
nebeneinander angezeigt werden. Werden mehr als zwei Aktionen benötigt, erhält
die Hauptaktion einen eigenen Button, während alle weiteren in ein
[ContextMenu](/03-components/actions/context-menu/overview) verschoben werden.

## AlertBadge und Badge

Die [Heading](/03-components/content/heading/overview) kann mit einem
[AlertBadge](/03-components/status/alert-badge/overview) oder einem
[Badge](/03-components/status/badge/overview) ergänzt werden. Dadurch wird der
User schnell über den Status oder über Metadaten des Inhalts einer Section
informiert.
