# Grundlagen

Die Component Table eignet sich besonders gut, wenn Datenmengen klar strukturiert, konsistent und leicht erfassbar dargestellt werden sollen.

## Best practices

Achte bei der Verwendung einer Table darauf, dass...

- die Anzahl der Spalten so gering wie möglich gehalten wird, damit das Scannen einfacher ist.
- Spaltenüberschriften kurz, eindeutig und prägnant formuliert sind.
- jede Zelle nur einen klar abgegrenzten Datenpunkt enthält – vermeide es, mehrere Informationen in einer Zelle zu bündeln.
- Zahlen, insbesondere Geldbeträge oder Zeitwerte, rechtsbündig angezeigt werden, um Vergleichbarkeit zu fördern.

## Verwendung

Verwende eine Table, um...

- große Datenmengen übersichtlich darzustellen und leicht vergleichbar zu machen.
- Informationen klar strukturiert in Zeilen und Spalten abzubilden (z. B. Userlisten, Rechnungsübersichten)
- Spaltenweise konsistente Inhalte (z. B. Preis, Status, Datum) schnell erfassbar zu machen.
