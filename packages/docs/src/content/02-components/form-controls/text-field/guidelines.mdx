TextFields ermöglichen es Nutzern, freien Text über das Keyboard einzugeben. Ein
erforderliches [Label](content/label) und eine optionale
[FieldDescription](form-controls/field-description) unterstützen dabei, die
Anforderungen an die Eingabe klar zu kommunizieren oder Fehlermeldungen
darzustellen. Ein TextField wird häufig in Forms, Dialogs oder
[Modals](overlays/modal) verwendet und bietet flexible Eingabeoptionen, um
verschiedene Anwendungsfälle abzudecken.

<InlineAlert>
### Best Practices
  <Content>
    - Frage nur nach Informationen, die wirklich notwendig sind.

    - Stelle sicher, dass das Label prägnant, vollständig sichtbar und leicht
    verständlich ist.

    - Verwende die FieldDescription sparsam und für sinnvolle
    Beispiele oder Formatierungshinweise.

    - Stelle sicher, dass Fehlermeldungen
    hilfreich sind, eine Lösung bieten und leicht umzusetzen sind.

    - Verwende
    keine Placeholder für wesentliche Informationen.

  </Content>
</InlineAlert>

## Verwendung

### Position

Das TextField wird meistens innerhalb eines [ColumnLayouts](column-layout)
platziert. Es füllt dabei jeweils die Breite des Columns, in dem es platziert
wird, vollständig aus. Dadurch folgt es auch einer vorhandenen Aufteilung in
Spalten, die das ColumnLayout eventuell mit sich bringt. Das ColumnLayout macht
es somit sehr einfach, übersichtliche Forms unter Berücksichtigung des Layouts
zu erstellen.

Am besten eignet sich ein 50% / 50% Raster für TextFields, sodass sie maximal
die Hälfte des Inhaltsbereichs einnehmen können. Das Column Layout kann außerdem
erweitert werden, um spezifischeren Layoutanforderungen gerecht zu werden. So
kann bspw. ein 66% / 33% Raster innerhalb einer Spalte des 50% / 50% Rasters
platziert werden, um Straßenname und Hausnummer zu einer sinnvollen Gruppe
zusammenzufassen.

## Inhalt

### Label

Jedes TextField muss ein zugehöriges [Label](content/label) besitzen. Das Label
ist entscheidend, um die Eingabeanforderungen zu verstehen und sollte daher so
gewählt werden, dass es alle Informationen liefert, um das TextField korrekt
ausfüllen zu können. Außerdem sollte das Label knapp (max. 2 Wörter), präzise
und weder versteckt noch auspunktiert sein. werden.

---

### FieldDescription

Die [FieldDescription](form-controls/field-description) kann als optionaler
Hilfstext unterhalb des TextFields angezeigt werden, um dem Nutzer ergänzenden
Informationen oder Anweisungen zu bieten. Es sollte sparsam verwendet werden und
kann:

- als zusätzliche Beschreibung verwendet werden, Eingabe- und
  Formatierungsbeispiele liefern oder
- eine Fehlermeldung darstellen, die den Benutzer alarmiert und Hilfestellung
  gibt, wenn die Feldanforderungen nicht erfüllt werden.

---

### Placeholder

Placeholder verursachen verschiedene Accessibility-Probleme. Sie verschwinden
beim Beginn der Texteingabe, werden von manchen Assistenztechnologien nicht
korrekt erkannt und sind kognitiv schwieriger zu verarbeiten, als bspw. eine
FieldDescription. \
Daher dürfen Placeholder keine Informationen enthalten, die für das korrekte Ausfüllen
des TextFields erforderlich sind.

Ergänze das TextField mit ...

- Einem Label für Informationen, die essenziell für das Verständnis der
  Eingabeanforderungen sind.
- Einer FieldDescription, um dem Nutzer Beispiele oder Formatierungshinweise zu
  geben.

## Verhalten

### Min / Max Width

Wird das TextField außerhalb eines ColumnLayouts verwendet, besitzt es keine
fixe Breite. Achte daher darauf, die Breite des TextFields angemessen zum Inhalt
zu wählen. So kann z.B. für ein TextField für eine URL deutlich mehr Platz
benötigt werden, als für eines, das nur eine Hausnummer abfragt.

---

## Accessibility

Das TextField kann sowohl über Tastaturnavigation erreicht und aktiviert werden
als auch von Assistenztechnologien angesteuert werden. Ein User kann außerdem im
TextField befindliche Icons auswählen, wenn diese interaktiv sind.

Damit auch der Inhalt barrierefrei ist, solltest du:

- Essenzielle Eingabeanforderungen über das Label kommunizieren.
- Unterstützung, Beispiele oder Formatierungshilfen über die FieldDescription
  bieten.
- Placeholder nicht für Inhalte verwenden, die zum Ausfüllen des TextFields
  hilfreich sind.
- Das Aria-Label nicht gepflegt werden, wenn die Komponente mit einem passenden
  Label verwendet wird.
